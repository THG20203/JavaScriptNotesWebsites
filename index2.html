FUNCTIONS

  • Function = piece of code we can re use. like variable but for chunks of code. Can call/ invoke functions as many times as we want.
  • When we write functions - we pass data into a function, functions can return data too. Can give us data back -> use for something else in the program. Return is for expressions or VALUES.
  • In function below we express something called parameters. Parameters are like variables that are specific to this function. They will get defined once we call/invoke the function.
    function fruitProcessor(apples, oranges) {
    const juice = `Juice with ${apples} apples and ${oranges} oranges`;
    return juice;
    }
    fruitProcessor(5, 0);
  • By passing in the real specific values, which will then get assigned to the parameters.
  • Actual values of parameters are called arguments.

FUNCTION DECLARATIONS/EXPRESSIONS

  • Function Declarations - simply use function keyword to declare a function
    function calcAge1(birthYear) {
    const age = 2037 - birthYear;
    return age;
    }
    executing function - calcAge1(1991);
    Parameter is the placeholder in the function, argument is actual value we use to fill in placeholder that is parameter.
  • Function Expressions - stored in variables, that variable wil then be/represent the function.
  • Expression -> produces value, store it in value.
    const calcAge2 = function(birthYear) {
    return 2037 - birthYear;
    }

ARROW FUNCTIONS

  • const calcAge3 = (birthYear) => 2024 - birthYear
  • Can ommit return statement if one line arrow function with now curly braces. But multiple line arrow functions with curly braces do need return statments.

INTRODUCTION TO ARRAYS

  • Data structure = arrays. Two most important data structures in JavaScript = arrays and objects
  • Arrays = big containers which we can put variables in a reference them. Data has to be stored, in data structures like arrays.
  • Array can hold as many values as we want and values of any type -> string, number, expression.
  • Arrays are 0 based - first element is number 0 work out how
  • How long the array is? arrayName.length.
    Length is a property, something we're going to talk about later in section when talk about objects. 'length' property is NOT 0 based, so won't be 2 even though 'Peter' is element.
  • const friends = ["Michael", "Steven", "Peter"]
  • Use length property to get last value in array.
    console.log(friends[friends.length - 1]);
  • Mutate array
    friends[2] = "Jay";
  • Only primitive values are immutable but array is NOT primitive value.
  • Can mutate arrays even though declared with const. What we can't do is replace entire array.

BASIC ARRAY OPERATIONS

  • Built in functions can directly apply on arrays - called array methods.
  • Push method = adds elements to end of array.
    arrayName.push("String");
    Push is a function, so returns a value, thats the length of the new array.
    const friends = ["Michael", "Steven", "Peter"];
    const newLength = friends.push("Jay");
    console.log(newLength); // returns '4' - the length of the array following adding with push
  • Unshift method = adds elements to beginning of array.
    unshift returns length of the array.
  • Pop method = removes last element of the array.
    Don't need to pass in argument
    arrayName.pop();
    Pop method also returns something but not length of new array, but instead it returns the removed element.
    const popped = friends.pop();
    console.log(popped); // Last element (Jay) saved into popped
  • Shift method = remove first element of array.
    friends.shift();
    Like pop, don't need arguments
    Once again returns element removed.
  • indexOf method const friends = ["Michael", "Steven", "Peter"];
    console.log(friends.indexOf("Steven")); // 1
    Why? Cause 0 index based.
    Try for element not there? Get -1
    console.log(indexOf("random name")); // -1
    You get -1
  • Inlcudes method = returns true if in array, false if not.
    console.log(friends.includes("Tristan")); // false

INTRODUCTION TO OBJECTS

  • Object = data structure. In objects, define key value pairs each of the keys is also called a property - so tristan object has 5 properties. Key is basically the variable name.
    const tristan = {
    firstName: "Tristan",
    lastName: "Griffiths",
    age: 2023 - 1997,
    job: "Software Developer",
    friends: ["Sam", "John", "Steven"], };
  • Array is ["Example", "Example"], Object is { key: value, key2: value2}
  • Order of values doesn't matter when retrieve them, unlike arrays. Get properties from object, using property name itself

DOT Vs. BRACKET NOTATION

  • Dot notation:
    console.log(jonas.lastName);
  • Bracket notation:
    console.log(jonas["lastName"]);
  • When do you use dot and when do you use bracket?
    When we need to first compute the property use [] bracket
    const nameKey = "Name"
    console.log(jonas["first" + nameKey]);
    OTHERWISE, use the dot notation.
  • Adding to objects? Very simple, just write new property name and value after equals for new key value pairing
    jonas.location = "Portugal";
    jonas["twitter"] = "@jonasschmedtmann";

OBJECT METHODS

  • Functions are a type of value, so can create key value pair where the value is just a function. Can add functions to objects.
    const tristan = {
    firstName: "Tristan",

    birthYear: 1997,
    job: "teacher",
