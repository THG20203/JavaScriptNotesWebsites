  • 7 Primitive data types:
    1. Number - floating point (meaning always have decimals, even if we don't see them or define them). Number are used for decimals and integers.

  • JavaScript has dynamic typing - we do not have to manually define data type of the value stored in a variable. Instead, data types are determined automatically.
  • JavaScript - value has a type. NOT the variable. Variables simply store a value that has as type.
  • Dynamic typing - we can assign a new value in a different data type to the same variable without a problem.

    Variable x can initially be a number then later a string. Can be useful but also difficult to find bugs.

  • Remember, variable declaration needs to happen before the log, because javascript is executed from top to bottom console.log(javascriptIsFun);

  • First time we declare a new varaible, we need to do it using let. But then, when we want to change the value - of the varaible, we simply write it again without the let.

LET, CONST AND VAR

  • Three ways of declaring variables in JavaScript -> let, const and var
  • let + const introduced ES6 -> modern JS. var is old way.
  • let keyword -> declare variables that will change later, during the execution of our program.
  • Shown example of let - reassigning value to a variable
    let age = 30;
    age = 31;
  • Use const - declare variables that are not supposed to change at any point in the future. Value in const variable cannot be changed.
  • Below WON'T work:
    const birthYear = 1991;
    birthYear = 1990;
  • This procudes another kind of javascript error, a TypeError.
  • The fact that variables created with const are immutable also means we cannot declare empty const variables.
  • const job; produces -> Syntax error: missing initializer in const declaration.
  • We should use const by default, let if things will change in the future. Good to have as little mutations as possible, because changing varialbes introduces potential for bugs.
  • let is blockscoped, var is function scoped
