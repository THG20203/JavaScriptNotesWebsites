VALUES AND VARIABLES

  • A value = piece of data, fundamental piece of information.
  • Value = smallest unit of information.
  • Values can be stored into variables, process is called declaring the variable - i.e. let firstName = "Jonas";
  • Outputting code to the browser = done into the console.log.
  • You pass the variable name console.log(firstName); into console. A Syntax error is error in writing the code.
  • Variable names can't start with numbers. Variable names can only contain numbers, letters or dollar sign.
  • Also can't use reserved keywords let new = 27; 'New' is a reserved keyword In javascript, don't start variable name with upper case letter.

    let me = {

    }
  • Example of primitive:
    let firstName= "Tristan";
    let age = 30; */
  • 7 Primitive data types:
    1. Number - floating point (meaning always have decimals, even if we don't see them or define them). Number are used for decimals and integers.
    2. String - sequence of characters used for text.
    3. Boolean - logical type that can only be true or false. It is used for taking decisions.
    4. Undefined - value taken by a variable that is not yet defined - empty value.
    5. Null - also means 'empty value'
    6. Symbol - (ES2015) - unique and cannot be changed (not useful now)
    7. BigInt - (ES2020) - larger integars than number type can hold
  • JavaScript has dynamic typing - we do not have to manually define data type of the value stored in a variable. Instead, data types are determined automatically.
  • JavaScript - value has a type. NOT the variable. Variables simply store a value that has as type.
  • Dynamic typing - we can assign a new value in a different data type to the same variable without a problem.
  • Example:
    Variable x can initially be a number then later a string. Can be useful but also difficult to find bugs.
  • Boolean is true or false.
  • Just writing 'true' is a boolean value - i.e. console.log(true). It is not a string cause printed in pink rather than white as a string would be printed in within the console.
  • We can store booleans in variables. let javascriptIsFun = true;
  • Remember, variable declaration needs to happen before the log, because javascript is executed from top to bottom console.log(javascriptIsFun);
  • In dynamic typing, what matters is that we do not write the let again.
    let javascriptIsFun = true;
    javascriptIsFun = "YES!";
  • First time we declare a new varaible, we need to do it using let. But then, when we want to change the value - of the varaible, we simply write it again without the let.
